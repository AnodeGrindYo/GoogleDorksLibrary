Collection de commandes de recherche avancées

Guide Complet de Construction des Google Dorks

Introduction

Les Google Dorks sont des opérateurs de recherche avancés qui permettent de trouver des informations spécifiques en utilisant la syntaxe de recherche de Google.

Opérateurs de Base

site: Recherche dans un site spécifique Ex: site:github.com password
filetype: Type de fichier spécifique Ex: filetype:pdf confidential
inurl: Mot dans l'URL Ex: inurl:admin login
intitle: Mot dans le titre Ex: intitle:"index of"
intext: Mot dans le texte Ex: intext:"password" filetype:txt
cache: Version en cache Ex: cache:example.com

Combinaisons Avancées

Recherche de fichiers sensibles

filetype:env "DB_PASSWORD" OR "SECRET_KEY" OR "API_KEY"

Pages d'administration

inurl:admin OR inurl:administrator OR inurl:wp-admin

Cloud storage exposés

site:s3.amazonaws.com "confidential" OR "backup"

Tableau des Opérateurs

Opérateur Description Exemple

filetype: Spécifie le type de fichier filetype:pdf
inurl: Recherche dans l'URL inurl:admin
intitle: Recherche dans le titre intitle:"login"
intext: Recherche dans le contenu intext:"password"
allintitle: Tous les mots dans le titre allintitle:admin login
allinurl: Tous les mots dans l'URL allinurl:admin login
OR Opérateur logique OU filetype:pdf OR filetype:doc
AND Opérateur logique ET password AND username
- Exclure un terme password -site:stackoverflow.com
" " Recherche exacte "internal use only"
* Joker "confidential * report"

Construction pas à pas

Étape 1: Définir l'objectif

Que recherchez-vous ? (ex: fichiers de configuration, données sensibles, etc.)

Étape 2: Choisir les opérateurs

Sélectionnez les opérateurs appropriés selon votre cible

Étape 3: Construire la requête

Combinez les opérateurs avec des mots-clés pertinents

Étape 4: Affiner

Ajoutez des filtres pour préciser vos résultats

Bonnes Pratiques

  • Respectez la légalité et l'éthique
  • Utilisez des tests sur vos propres systèmes
  • Signalez les vulnérabilités aux propriétaires
  • Ne téléchargez pas de fichiers sensibles
  • Documentez vos découvertes de manière responsable

Exemples Pratiques

Recherche de clés API

filetype:env "API_KEY=" OR "SECRET_KEY=" OR "PRIVATE_KEY="

Pages d'admin ouvertes

intitle:"index of" "admin" OR "administrator" OR "login"

Données clients exposées

filetype:csv OR filetype:xlsx "customer" "email" OR "phone"

Configuration Docker

filetype:Dockerfile "ENV" "PASSWORD" OR "SECRET" OR "TOKEN"
